# Quick Follow アルゴリズムフロー

## カメラスキャン

1. `getUserMedia` でカメラ映像を開始する。
2. MediaPipe の物体検出器を `VIDEO` モードで準備する。
3. Tesseract の OCR worker を準備する。
4. 750ms ごとに1回だけ解析サイクルを実行する。
5. 現在のカメラフレームから物体を検出する。
6. `label` に `cell phone`、`mobile phone`、`phone` のいずれかを含む検出だけを残す。
7. スマホが見つからない場合は、そのままスキャンを継続する。
8. スマホ候補が複数ある場合は、最も大きいバウンディングボックスを採用する。
9. 現在のカメラフレームを内部用の静止画キャンバスへコピーする。
10. 静止画キャンバスから検出されたスマホ領域を切り出す。
11. スマホ画像の上部 50% を切り出す。
12. 上部プロフィール領域に対して OCR を実行する。
13. 有効な X ID が見つからない場合は、画面を固定せず次の 750ms サイクルで再試行する。
14. 有効な X ID が見つかった場合は、スキャンとカメラストリームを停止する。
15. 採用した静止フレームをプレビューに表示する。
16. プレビューを検出済みスマホ領域へズームする。
17. 検出した ID の文字領域を黄色で囲う。
18. 検出した ID と「開く」ボタンを確認モーダルに表示する。

## 画像インポート

1. 選択された画像ファイルをキャンバスに読み込む。
2. 元画像をプレビューに表示する。
3. MediaPipe の物体検出器を `IMAGE` モードで準備する。
4. 画像キャンバスから物体を検出する。
5. `label` に `cell phone`、`mobile phone`、`phone` のいずれかを含む検出だけを残す。
6. スマホが見つからない場合はエラーを表示する。
7. スマホ候補が複数ある場合は、最も大きいバウンディングボックスを採用する。
8. プレビューを検出済みスマホ領域へズームする。
9. 画像キャンバスから検出されたスマホ領域を切り出す。
10. スマホ画像の上部 50% を切り出す。
11. 上部プロフィール領域に対して OCR を実行する。
12. 有効な X ID が見つからない場合はエラーを表示する。
13. 有効な X ID が見つかった場合は、検出した ID の文字領域を黄色で囲う。
14. 検出した ID と「開く」ボタンを確認モーダルに表示する。

## OCR と ID 判定

1. OCR 対象は、切り出したスマホ領域の上部 50% のみとする。
2. OCR はまず元の切り出し画像に対して実行し、必要に応じて前処理済み画像でも実行する。
3. 前処理ではグレースケール化とコントラスト強調を行う。
4. OCR は英語設定で実行し、対象文字を X ID に使える文字へ制限する。
5. OCR 結果を行単位に分割する。
6. 文字領域の座標は、symbol の bbox、word の bbox、line の bbox の順で採用する。
7. 行全体が `@[A-Za-z0-9_]{1,15}` に一致する場合だけ ID 候補にする。
8. ID 候補は、検出された文字の高さが小さいほど、かつ左にあるほど高く評価する。
9. 最もスコアが高い候補を最終 ID とする。
10. 最終 URL は `https://x.com/{@を除いたID}` とする。
